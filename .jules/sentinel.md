## 2024-05-24 - URL Validation and Sanitization Bypasses (SSRF/XSS)
**Vulnerability:** The application used `.includes('evzone.com')` to validate and sanitize URLs, and lacked an explicit protocol allowlist for `sanitizeUrl`. This allowed attackers to craft URLs like `https://evzone.com.attacker.com` to bypass domain validation, and `javascript:alert(1)` to bypass sanitization since assigning `parsed.protocol = 'https:'` fails silently on special schemes.
**Learning:** `.includes()` is extremely insecure for domain validation. Furthermore, the `URL` API's behavior of silently failing to modify the protocol of non-standard/special schemes like `javascript:` means sanitizers must strictly enforce an allowlist and reject unauthorized protocols, rather than attempting to "fix" them.
**Prevention:** Always use strict domain matching (`===` or `.endsWith()`) when checking hostnames. Always enforce an allowlist of permitted protocols (e.g., `['http:', 'https:']`) and explicitly return a safe fallback or error if the protocol is unauthorized.
