## 2024-05-24 - URL Sanitizer SSRF & XSS Bypass via Protocol Mutation Failure
**Vulnerability:** The application used `.includes('evzone.com')` for URL domain validation, allowing SSRF/Open Redirects via subdomains like `evzone.com.attacker.com`. Additionally, attempting to enforce HTTPS by modifying `url.protocol = 'https:'` fails silently for special browser schemas (like `javascript:`), enabling XSS vulnerabilities by passing dangerous protocols through the sanitizer untouched.
**Learning:** The `URL` API's `.protocol` setter ignores attempts to overwrite special schemas (`javascript:`, `data:`, etc.) if the new protocol is not a standard one, or silently maintains the schema, thus failing closed in an insecure way. Domain validation using `.includes()` is extremely risky as it matches substrings anywhere in the hostname.
**Prevention:** Always use strict domain validation with exact matching (`===`) or exact suffix matching (`.endsWith()`) alongside a leading dot (e.g., `.endsWith('.evzone.com')`). When sanitizing URLs, validate the protocol explicitly against an allowlist (e.g., `['http:', 'https:']`) and reject unapproved inputs early instead of relying solely on mutation.
