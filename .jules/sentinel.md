## 2024-05-24 - URL Sanitizer bypass via include and unchecked protocol mutation
**Vulnerability:** `isValidUrl` used `.includes('evzone.com')` to validate domains, making it vulnerable to SSRF/Open Redirect (e.g. `evzone.com.attacker.com`). `sanitizeUrl` attempted to upgrade HTTP to HTTPS but didn't restrict protocols, so schema manipulations failed silently for `javascript:` URLs, leading to an XSS bypass.
**Learning:** `.includes()` is extremely dangerous for hostname validation and should be avoided in favor of exact match (`===`) or exact subdomain match (`.endsWith('.domain.com')`). Furthermore, modifying `url.protocol` is not an adequate sanitization strategy without an explicit allowlist (like `http:` and `https:`), as unapproved protocols will fail to mutate and be returned unaltered.
**Prevention:** Always use strict exact matching or endsWith for domain validation. Always use an explicit allowlist to reject unapproved protocols before applying any protocol mutation or validation logic.
