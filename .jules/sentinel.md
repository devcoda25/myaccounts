## 2024-05-18 - [CRITICAL] Fix URL Sanitization XSS Vulnerability
**Vulnerability:** The application was using an insecure method (`.includes`) to validate domain names in URLs, allowing attackers to bypass validation using domains like `evzone.com.attacker.com`. Additionally, dynamic URLs fetched from the API and injected into `href` attributes were not sanitized to prevent malicious protocols like `javascript:`, exposing the application to Stored Cross-Site Scripting (XSS).
**Learning:** `.includes` is insufficient for domain validation as it matches substrings anywhere in the hostname. URLs derived from external sources or APIs must always be sanitized before being rendered in `href` or `src` attributes, even if the API is considered trusted, to maintain defense-in-depth.
**Prevention:** Always use exact matching or `.endsWith` (with appropriate dot prefixes) to validate trusted domains. Explicitly block dangerous non-HTTP/HTTPS protocols in URL sanitizers to prevent XSS. Enforce the use of a robust `sanitizeUrl` function for any dynamically rendered URLs.
