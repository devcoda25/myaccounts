## 2024-06-13 - [Fix Open Redirect Bypass in URL Validation]
**Vulnerability:** The `isValidUrl` sanitizer in `src/sanitizers/url.ts` used `.includes('evzone.com')` to validate domains, which allowed attackers to bypass the check using domains like `https://evzone.com.attacker.com` or `https://attacker-evzone.com`. Additionally, an unprotected redirect sink in `src/features/auth/continue/Index.tsx` relied on unvalidated user-supplied `ctx.redirectUri` values.
**Learning:** Using substring matching (`.includes()`) for domain validation is fundamentally insecure and vulnerable to SSRF and Open Redirect bypasses. Furthermore, when blocking substrings like `//` to prevent protocol-relative bypasses, care must be taken not to block valid URLs containing those sequences in query parameters (e.g., `?redirect_uri=https://`).
**Prevention:** Always use strict exact matching (`===`) or proper subdomain suffix matching (`.endsWith('.domain.com')`) for domain validation. Ensure all redirect sinks (e.g., `window.location.href`) are wrapped with robust URL validation.
