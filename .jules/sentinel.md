## 2026-04-03 - [XSS via javascript: URL Bypass in sanitizeUrl]
**Vulnerability:** The `sanitizeUrl` function in `src/sanitizers/url.ts` used the `URL` object to parse input but only checked `parsed.protocol !== 'https:'` to upgrade it. It silently passed dangerous schemas like `javascript:` because they weren't explicitly blocked, leading to XSS vulnerabilities. Additionally, it broke valid relative URLs by failing the `URL` constructor check.
**Learning:** The JavaScript `URL` API fails silently when attempting to mutate the protocol for special schemas like `javascript:`. URL sanitizers must explicitly validate against an allowlist (e.g., `http:`, `https:`) and reject unapproved protocols instead of attempting mutation. Relative URLs must be handled gracefully to avoid breaking application functionality while preventing protocol-relative (`//`) bypasses.
**Prevention:** Always use explicit protocol allowlists in URL sanitizers. Ensure robust testing for edge cases like `javascript:` schemas and relative paths.
