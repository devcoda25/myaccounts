## 2025-02-21 - Fix URL Validation and Sanitization Bypasses
**Vulnerability:** URL validation and sanitization could be bypassed due to weak substring matching (`.includes()`) and reliance on the URL object's protocol setter. `sanitizeUrl` attempted to override `protocol` to `https:` for arbitrary protocols, but this fails silently for special schemas like `javascript:`. `isValidUrl` used `.includes()` which allowed attacker domains like `evzone.com.attacker.com` to match `evzone.com`.
**Learning:** The JS `URL` API behaves non-intuitively: setting `url.protocol = 'https:'` when `url.protocol` is `javascript:` has no effect. Additionally, using `String.prototype.includes()` for domain or path validation creates SSRF/Open Redirect vulnerabilities by matching sub-strings within attacker-controlled strings.
**Prevention:** Always validate protocols against a strict allowlist (e.g., `['http:', 'https:']`) instead of attempting to mutate an unknown protocol. Use exact matching (`===`) or strict suffixes (`.endsWith()`) against an array of trusted domains rather than `String.prototype.includes()`.
