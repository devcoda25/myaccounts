## 2026-03-11 - [CRITICAL] Fix URL validation bypass (SSRF/Open Redirect & XSS)
**Vulnerability:** The `isValidUrl` and `sanitizeUrl` functions in `src/sanitizers/url.ts` used insecure `.includes()` checks for hostname validation (e.g., `parsed.hostname.includes('evzone')`), allowing attackers to bypass validation using domains like `evzone.com.attacker.com` or `attackerevzone.com`. Furthermore, `sanitizeUrl` failed to explicitly block non-HTTP/HTTPS protocols, allowing XSS payloads like `javascript:alert(1)` to be returned verbatim.
**Learning:** Using `.includes()` for domain validation is inherently insecure. Attackers can easily register subdomains or domain variations that match the string. Additionally, URL sanitizers must use a strict allowlist for protocols (e.g., `http:`, `https:`) and fail closed (return empty) for anything else.
**Prevention:** Always use exact matching (`===`) or `.endsWith('.domain.com')` when validating hostnames. Ensure sanitizers explicitly restrict allowed URL schemas and fail securely when an unsupported schema is encountered.
