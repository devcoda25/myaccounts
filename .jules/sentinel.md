## 2024-05-18 - Prevent XSS & SSRF via Strict Domain & Protocol Validation in `sanitizeUrl`
**Vulnerability:** URL sanitization allowed unapproved protocols (like `javascript:`) by assuming changing the protocol would work. In JavaScript, modifying `new URL('javascript:alert(1)').protocol = 'https:'` silently fails and leaves `javascript:`. Furthermore, domain validation using `.includes('evzone.com')` was vulnerable to SSRF and Open Redirect attacks (e.g. `evzone.com.attacker.com`).
**Learning:** The URL API's `protocol` assignment fails silently for special schemas like `javascript:`. Do not try to mutate invalid schemas; reject them explicitly using an allowlist (`['http:', 'https:']`). For domain validation, strictly use `===` and `.endsWith()` instead of `.includes()`.
**Prevention:** Always validate protocols against an explicit allowlist and enforce strict domain-suffix matching. Do not trust mutations on the `URL` object when the original protocol is unapproved.
