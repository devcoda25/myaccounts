
## 2025-02-17 - Fix URL validation bypass (SSRF/Open Redirect) & XSS in URL sanitizers
**Vulnerability:** URL validation and sanitization functions (`isValidUrl` and `sanitizeUrl`) used weak `.includes('evzone.com')` checks for domain validation, making them susceptible to bypasses (e.g. `evzone.com.attacker.com`). Furthermore, `sanitizeUrl` attempted to mutate `parsed.protocol = 'https:'` on unauthorized schemas (e.g., `javascript:`), but the `URL` API silently fails to mutate the protocol on special schemas like `javascript:`, exposing the application to XSS.
**Learning:** Checking for substrings in domain names is a common anti-pattern that leads to Open Redirect and SSRF bypasses. The JavaScript `URL` API's silent failure when modifying protocols for special schemas means sanitizers must explicitly whitelist acceptable protocols (`http:`, `https:`) and reject unapproved protocols instead of mutating them.
**Prevention:** Always use strict exact matching (`===`) or suffix matching (`.endsWith()`) against a list of trusted domains. Validate the URL protocol against an explicit allowlist (e.g., `['http:', 'https:'].includes(parsed.protocol)`) instead of trying to mutate unapproved schemas.
