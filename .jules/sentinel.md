## 2024-11-20 - Strict Domain Validation and Protocol Rejection in URL Sanitizer
**Vulnerability:** Weak URL parsing and validation where SSRF/Open Redirect can be triggered by `.includes('evzone.com')` (e.g. `evzone.com.attacker.com`). Additionally, XSS is possible when passing `javascript:` to `sanitizeUrl()`, which attempts to coerce the protocol via `parsed.protocol = 'https:'`. This fails silently for `javascript:` schemas, returning the malicious URL instead.
**Learning:** `URL.protocol` modification fails silently when transitioning to/from special schemas (like `javascript:`). Attempting to "upgrade" HTTP-like protocols should only be done *after* strictly validating the schema is HTTP/HTTPS. Additionally, domain validation must rely on exact matches or strict suffix checks (`.endsWith`) to prevent bypasses.
**Prevention:** Reject unapproved protocols early in URL sanitizers instead of attempting assignment coercion. Implement a strict array lookup (`.endsWith` / exact match) for domain validation to ensure robust security against SSRF and Open Redirect vulnerabilities.
