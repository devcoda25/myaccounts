## 2024-05-18 - [Critical] Open Redirect / SSRF in URL Validation
**Vulnerability:** URL validation logic used a weak `.includes('evzone.com')` check on domain names, which is easily bypassed by domains like `evzone.com.attacker.com`. Additionally, `window.location.href` was set to unvalidated `redirectUri` values from URL search parameters.
**Learning:** Checking for substring matches in domain names for validation allows simple bypasses. Also, modifying protocol via the Javascript `URL` API (e.g. `parsed.protocol = 'https:'`) fails silently for certain schemas like `javascript:`, defeating sanitation attempts if an allowlist is not used.
**Prevention:** Always validate domains using strict `.endsWith('.evzone.com')` or exact `=== 'evzone.com'` matching. URL sanitizers must explicitly validate against an allow-list of schemas (e.g., `['http:', 'https:']`) and reject unapproved schemas instead of attempting mutation. Ensure any dynamic assignment to `window.location.href` uses properly validated URLs.
