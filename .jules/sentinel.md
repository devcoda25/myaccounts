
## 2025-05-01 - URL validation SSRF & protocol mutation bypass
**Vulnerability:** `isValidUrl` used `parsed.hostname.includes('evzone.com')`, which allowed attacker domains like `evzone.com.attacker.com` (SSRF/Open Redirect). Additionally, `sanitizeUrl` successfully parsed schemas like `javascript:alert(1)` without throwing an error, but attempted to mutate `parsed.protocol = 'https:'`. This protocol mutation fails silently on special schemas (like `javascript:`), bypassing the sanitization and causing XSS. Furthermore, `new URL()` throws an error on safe relative paths, which were being stripped unintentionally.
**Learning:** Hostname matching must use exact matches (`===`) or strict suffixes (`.endsWith()`). Modifying the `protocol` property on a `URL` object is not a reliable way to sanitize malicious schemas because it fails silently for special URLs.
**Prevention:** Always use exact matching or `.endsWith()` for domain validation. For URL sanitization, strictly validate the `parsed.protocol` against an explicit allowlist (e.g., `['http:', 'https:', 'mailto:', 'tel:']`) and reject unapproved protocols immediately, rather than attempting to mutate them. Explicitly catch relative paths in the `catch` block by verifying they begin with `/` but not `//` or `/\\`.
