## 2024-05-15 - URL Domain Validation Bypass & Protocol Mutation Failure
**Vulnerability:** The URL sanitizer `isValidUrl` used `parsed.hostname.includes('evzone.com')` to validate domains, which allowed bypasses like `evzone.com.attacker.com`. Additionally, `sanitizeUrl` attempted to upgrade `url.protocol` to HTTPS on potentially dangerous schemes like `javascript:`.
**Learning:** The JavaScript `URL` API silently fails when trying to reassign the `.protocol` property on some special schemas (like `javascript:`), meaning an attempt to make it "https:" would do nothing and return the original `javascript:alert(1)` schema, resulting in an XSS payload bypassing sanitization. Furthermore, domain validation must never use `.includes()`.
**Prevention:** Always use strict exact matching (`===`) or suffix matching (`.endsWith('.evzone.com')`) for domain validation to prevent Open Redirects or SSRF bypasses. Always fail securely in sanitizers by validating and returning an empty string if the protocol is not explicitly in an allowlist like `['http:', 'https:']` rather than mutating unknown or dangerous schemes.
