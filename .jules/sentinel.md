## 2025-03-25 - [HIGH] URL Validation Bypass and Protocol Injection via `includes()`
**Vulnerability:** URL validation and sanitization using `.includes('evzone.com')` could be trivially bypassed by an attacker registering subdomains like `evzone.com.attacker.com`. Additionally, `sanitizeUrl` did not validate initial protocols, meaning input like `javascript://evzone.com/%0Aalert(1)` successfully bypassed the filter, allowing XSS because modifying `URL.protocol` silently fails for special schemas like `javascript:`.
**Learning:** `includes()` is dangerous for validating domains because it matches anywhere in the string. Furthermore, JavaScript's URL object fails silently when attempting to mutate `.protocol` for special schemas (like `javascript:`). Thus, sanitizers must explicitly validate against an allowlist and reject unapproved protocols instead of mutating them.
**Prevention:** Always use exact matching (`===`) or `.endsWith('.evzone.com')` for validating domains, and strictly validate URL protocols (e.g., `['http:', 'https:'].includes(parsed.protocol)`) instead of relying on modifying the `protocol` attribute.
