## 2024-05-09 - Fix XSS, SSRF and Open Redirect Bypass in URL sanitizers
**Vulnerability:** The domain validation using `.includes()` allowed domain bypassing (e.g. `evzone.com.attacker.com`). Additionally, `sanitizeUrl` relied on `URL.protocol` mutability but silently failed on `javascript:` schemes, allowing XSS.
**Learning:** `.includes()` on domain names opens up SSRF and Open Redirect bypass vectors. URL schema mutability failures can lead to XSS when an explicit protocol allowlist isn't checked before returning the serialized URL string. Fallbacks for relative paths during `URL` parsing errors must ensure they cannot become protocol-relative paths (`//`) to prevent Open Redirect.
**Prevention:** Always use strict exact matching (`===`) or suffix checking (`.endsWith()`) when validating domain constraints. Use a strict explicit allowlist logic (`http:`, `https:`) for evaluating URL scheme protocols and explicitly fail/reject unapproved ones rather than relying on mutating the protocol. Reject `//` explicitly on relative fallbacks.
