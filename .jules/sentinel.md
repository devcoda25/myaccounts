## 2024-03-28 - Insecure Random Number Generation Fallback
**Vulnerability:** The `safeRandomBytes` function and `generateId` function in `src/utils/helpers.ts` used an insecure fallback `Math.random()` to generate random bytes and IDs when the cryptographically secure method `window.crypto.getRandomValues` failed or wasn't available. This weak random generation could allow attackers to predict random values, making sensitive applications (like OTPs or Passkeys) susceptible to brute-force or prediction attacks.
**Learning:** We need to explicitly fail securely by throwing an error rather than silently falling back to a weak PRNG. `Math.random()` should never be used for security-critical random generation.
**Prevention:** Avoid the use of `Math.random` fallback for anything resembling cryptographically secure random bytes or IDs. Ensure error messages clearly indicate when secure RNG is not supported, forcing developers or users to run in a secure context rather than having a false sense of security. Always use `window.crypto.getRandomValues`.
