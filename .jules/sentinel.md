## 2024-04-24 - Fix SSRF, Open Redirect, and XSS Bypasses in URL Validation and Sanitization
**Vulnerability:** The `isValidUrl` function used `.includes('evzone.com')` to validate domains, allowing bypasses using domains like `evzone.com.attacker.com`. Additionally, `sanitizeUrl` relied on modifying `url.protocol` without an explicit check; for schemas like `javascript:`, setting `url.protocol` to `https:` silently fails without an error, bypassing sanitization and leading to XSS if the URL is dynamically rendered into an anchor tag or script source.
**Learning:** In JavaScript's URL API, modifying `url.protocol` behaves differently for certain protocols (like `javascript:`) and will silently fail instead of changing the protocol. Additionally, string methods like `.includes()` are insufficient for domain validation as they do not distinguish subdomains from completely different domains containing the substring.
**Prevention:** URL validation functions must use exact domain matching (`===`) or strict subdomain suffix matching (`.endsWith('.domain.com')`) against a trusted domain allowlist. Sanitizers must explicitly validate against an allowlist of approved protocols (`['http:', 'https:']`) and reject unapproved protocols immediately instead of attempting to blindly mutate the string.
