## 2024-05-24 - [Insecure URL Sanitization]
**Vulnerability:** The custom URL sanitization logic (`isValidUrl` and `sanitizeUrl` in `src/sanitizers/url.ts`) used `.includes()` for hostname validation (e.g. `parsed.hostname.includes('evzone.com')`), which could easily be bypassed with an attacker-controlled domain like `evzone.com.attacker.com`. Additionally, `sanitizeUrl` allowed dangerous protocols (like `javascript:`) by not explicitly enforcing an allowlist of valid protocols before transforming or returning the URL.
**Learning:** Checking for domain inclusion using `.includes()` is dangerous because it allows substrings anywhere in the domain name. Always perform strict equality checks (`===`) for exact domains or `.endsWith('.domain.com')` to encompass subdomains accurately. Also, explicitly restricting allowable protocols (like `http:` and `https:`) prevents protocol-based XSS attacks.
**Prevention:** In functions designed to sanitize or validate URLs, enforce a rigid allowlist of safe protocols (e.g. `['http:', 'https:']`) immediately upon parsing. Validate the `hostname` property using strict equality or `.endsWith` checks, never simple substring searches.
