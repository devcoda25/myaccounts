## 2025-03-05 - URL Protocol Mutation Failure and Hostname Bypass
**Vulnerability:** The application was vulnerable to SSRF/Open Redirects due to using `.includes()` to validate hostnames (e.g., `evzone.com` could match `evzone.com.attacker.com`). Furthermore, an XSS vulnerability existed because the `sanitizeUrl` function attempted to mutate special protocols like `javascript:` to `https:` using `parsed.protocol = 'https:'`. The JavaScript `URL` API silently fails when attempting to mutate such special schemas, allowing the `javascript:` payload to persist in the returned URL.
**Learning:** The JavaScript `URL` API behaves unexpectedly when mutating the `protocol` property of special schemas. Hostname validation using string inclusion is fundamentally flawed.
**Prevention:** Always use strict exact matching (`===`) or suffix matching (`.endsWith()`) for hostnames. Never attempt to mutate URL protocols to "fix" them; instead, strictly allowlist safe protocols (`http:`, `https:`) and reject unapproved protocols immediately.
