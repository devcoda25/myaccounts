
## 2024-11-20 - Fix URL Sanitization and Validation Bypass
**Vulnerability:** The application was vulnerable to Open Redirects, Server-Side Request Forgery (SSRF) and Cross-Site Scripting (XSS) due to weak URL validation. `isValidUrl` used `hostname.includes('evzone.com')`, allowing bypasses like `https://evzone.com.attacker.com`. Additionally, `sanitizeUrl` failed to strictly validate URL protocols (allowing schemas like `javascript:`) and used `.includes('evzone')` for domain checks, potentially upgrading malicious unauthenticated domains to HTTPS while failing to reject harmful URIs entirely.
**Learning:** Using `.includes()` for hostname validation allows partial matches on subdomains or arbitrary strings managed by attackers. Furthermore, modifying a URL property using the `URL` API (like `parsed.protocol = 'https:'`) can fail silently when the schema is a non-standard protocol like `javascript:`, bypassing attempts to sanitize it to HTTPS without blocking the protocol initially.
**Prevention:** Always validate domains using exact matches (`===`) or strict suffix matching (`.endsWith('.domain.com')`). URL sanitizers must explicitly validate against an allowlist of permitted protocols (e.g., `['http:', 'https:']`) and reject unapproved protocols immediately instead of attempting mutation.
