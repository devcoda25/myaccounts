## 2024-05-24 - [Strict URL Validation and XSS Prevention]
**Vulnerability:** URL validation routines (`isValidUrl`) used a loose `.includes('evzone.com')` match which is susceptible to bypasses (e.g. `evzone.com.evil.com`). Furthermore, the `sanitizeUrl` utility did not explicitly enforce a safe schema whitelist (e.g., `http:`, `https:`), leaving dynamic API-driven links (like `app.url` in the Dashboard) open to `javascript:` based XSS attacks when rendered in `href` properties.
**Learning:** React prevents XSS natively in text nodes, but properties like `href` or `src` are common vectors if they reflect unsanitized external strings directly. Always enforce strict whitelist approaches for protocol checks instead of partial host matching.
**Prevention:** Rely on exact matching (`===`) or `endsWith('.domain.com')` for URL hosts. Implement strict protocol-level enforcement discarding any URI strings originating outside of a `http:`/`https:` namespace before passing them to the UI.
