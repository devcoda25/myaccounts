## 2024-05-24 - Unvalidated URL Redirection (Open Redirect & XSS)
**Vulnerability:** User-provided URLs (e.g., via `redirect_uri` query parameters) were passed directly to `window.location.href` assignments without validation. This allowed an attacker to supply malicious protocols like `javascript://evzone.com/%0Aalert(1)` to achieve Cross-Site Scripting (XSS), or perform an Open Redirect to an untrusted domain.
**Learning:** Even when the intent is to redirect back to a "first-party app", checking domains or using simple host extraction (like `new URL(url).host`) is insufficient if the redirect assignment allows arbitrary schemas (e.g., `javascript:`, `vbscript:`, `data:`). Furthermore, the URL API might preserve the `javascript:` schema silently when attempting to mutate it.
**Prevention:** Always validate user-provided URLs using strict allowlists for protocols (e.g., `https:`) and authorized domains before assignment to `window.location.href`. In this application, use the established `isValidUrl` sanitization utility to ensure redirects only point to trusted, safe destinations.
