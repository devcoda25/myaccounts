## 2024-05-16 - [CRITICAL] Prevent Open Redirect and XSS via URL manipulation
**Vulnerability:** Weak URL validation (`.includes('evzone.com')`) in `isValidUrl` allowed bypasses via subdomains (e.g. `evzone.com.attacker.com`). Weak URL protocol handling in `sanitizeUrl` attempted to override `protocol` if not `https:`, which silently fails for special schemes like `javascript:`, enabling XSS in components like `<Paper component="a">`.
**Learning:** `String.prototype.includes` is insufficient for domain validation against SSRF and Open Redirects. `URL` objects do not allow protocol reassignment from special schemes (like `javascript:`) to standard schemes, silently ignoring the change and preserving the malicious protocol.
**Prevention:** Always use strict regex or exact string matching for domains. For URLs, validate the protocol strictly against an allowlist (e.g., `['http:', 'https:', 'mailto:', 'tel:']`) rather than attempting to mutate or normalize malicious schemes.
