## 2024-05-14 - Fix XSS and Open Redirect in URL Sanitizers
**Vulnerability:** URL sanitization relies on `new URL()` but fails to validate the parsed protocol (e.g., allows `javascript:alert(1)//evzone.com`), leading to XSS. Additionally, domain validation used `.includes('evzone.com')` which allowed bypasses like `evzone.com.attacker.com` leading to Open Redirect or SSRF.
**Learning:** When using the JavaScript `URL` constructor, parsing malicious URLs like `javascript:alert(1)//domain.com` does not throw an error; it assigns a `javascript:` protocol and an empty hostname, circumventing naive hostname validation if the protocol is not first strictly checked against an allowlist.
**Prevention:** Always enforce a strict protocol allowlist (`['http:', 'https:', 'mailto:', 'tel:']`) when sanitizing URLs. Use strict regular expressions (e.g., `^([a-z0-9-]+\.)*(evzone\.com|evzone\.app|evzonemarketplace\.com)$/i`) rather than `.includes()` to validate domains to prevent SSRF and Open Redirect bypasses.
