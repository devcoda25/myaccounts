## 2024-03-28 - [CRITICAL] URL Validation and Sanitization Bypasses via URL API
**Vulnerability:** The codebase used `parsed.hostname.includes('evzone.com')` for URL domain validation, allowing bypasses via open redirect or SSRF (e.g., `https://evzone.com.attacker.com`). Additionally, `sanitizeUrl` attempted to upgrade HTTP to HTTPS using `parsed.protocol = 'https:'` but this mutator silently fails in the JavaScript `URL` API for special schemas like `javascript:`. If a user inputted `javascript://evzone.com/%0Aalert(1)`, it would bypass checks, silently fail to upgrade the protocol, and return a malicious XSS payload intact.
**Learning:** Never use `.includes()` for domain/hostname validation. The `URL` API's mutator for `protocol` will fail silently if mutating to/from certain special schemas (such as `javascript:`), which can lead to returning dangerous un-sanitized output.
**Prevention:** Always use exact matching (`===`) and strict suffix matching (`.endsWith('.evzone.com')`) for hostname validation. Instead of mutating arbitrary URL schemas to secure ones, implement a strict explicit allowlist of safe schemas (e.g., `['http:', 'https:'].includes(parsed.protocol)`) and reject/drop the URL entirely if the initial schema is unapproved.
