## 2025-05-12 - Fix Open Redirect / SSRF in URL validation
**Vulnerability:** The `isValidUrl` function utilized `parsed.hostname.includes('evzone.com')` to validate redirect URIs. This implementation was vulnerable to Open Redirect and Server-Side Request Forgery (SSRF) bypasses because an attacker could register domains like `evzone.com.attacker.com` or `not-evzone.com` which successfully satisfied the `.includes()` string comparison.
**Learning:** Naive substring matching (like `String.prototype.includes`) should never be used for validating network origins or hostnames, as it inherently fails to properly segment domain boundaries. The JavaScript `URL` constructor correctly extracts the `hostname` (e.g., `evzone.com.attacker.com`), but weak validation nullifies that structured extraction.
**Prevention:** Always validate hostnames and origins using strict exact matching (e.g., `hostname === "trusted.com"`), strict suffix matching (e.g., `hostname.endsWith(".trusted.com")`), or anchored, robust regular expressions (e.g., `/^([a-z0-9-]+\.)*trusted\.com$/i`) that explicitly verify domain boundaries to prevent unauthorized prefixing or suffixing bypasses.
