## 2024-11-06 - Insecure Randomness Fallback in Core Utilities
**Vulnerability:** Several features (e.g. `safeRandomBytes`, passkeys) were relying on insecure `Math.random()` to generate bytes if the Web Crypto API `window.crypto` was unavailable. Using `Math.random()` for security tokens or challenges (like WebAuthn challenges or secure session IDs) produces predictable outputs, opening the door to replay and token prediction attacks.
**Learning:** Utilities intended for cryptographic contexts must strictly fail-closed. Attempting graceful degradation by falling back to non-cryptographic `Math.random()` introduces subtle but severe security flaws, subverting the security goals of the utility without alerting the caller.
**Prevention:** Always throw an explicit error when `window.crypto.getRandomValues()` is unavailable in `safeRandomBytes` or equivalent utilities. Features must catch these exceptions at the call site and handle the error UX appropriately instead of relying on insecure bytes.
