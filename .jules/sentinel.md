## 2024-10-27 - [CRITICAL] Prevented URL Validation Bypass and XSS in sanitizers
**Vulnerability:** The `isValidUrl` function used `parsed.hostname.includes('evzone.com')`, enabling attackers to bypass validation using subdomains (e.g., `evzone.com.attacker.com`) to conduct Server-Side Request Forgery (SSRF) and Open Redirects. Furthermore, the `sanitizeUrl` function relied on mutating `parsed.protocol` to upgrade HTTP to HTTPS, but this fails silently for special protocols like `javascript:`, exposing an XSS vulnerability when user inputs are injected into DOM attributes like `href`.
**Learning:** URL hostnames must be strictly matched using exact match (`===`) or exact suffix matching (`endsWith`) against an explicit allowlist. When working with the native `URL` class, do not rely on mutating `protocol` as a sanitizer because assigning to `url.protocol` fails silently for disallowed schemas like `javascript:`.
**Prevention:** Always validate domains against strict suffix checks and enforce explicit protocol allowlists (e.g., `['https:', 'http:']`) by rejecting, not mutating, invalid protocols.
