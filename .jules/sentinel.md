## 2024-03-27 - [URL Validation SSRF/XSS]
**Vulnerability:** URL validation (`isValidUrl`) and sanitization (`sanitizeUrl`) in `src/sanitizers/url.ts` relied on `.includes('evzone.com')`, which allows bypassing checks using subdomains on attacker-controlled domains (e.g., `evzone.com.attacker.com`). Furthermore, `sanitizeUrl` attempted to overwrite the protocol of `javascript:` URLs to `https:`, which silently fails in the `URL` API, returning the unmodified `javascript:` string, leading to XSS when rendering dynamic `<Paper component="a" href="...">` elements.
**Learning:** The `URL` interface in JS silently ignores protocol assignment if the original protocol is special (like `javascript:`). Using `.includes()` on hostnames is highly vulnerable to SSRF and open redirect bypasses.
**Prevention:** Always validate hostnames using exact matching (`===`) or exact suffix matching (`.endsWith('.domain.com')`). Never attempt to mutate unapproved protocols; instead, explicitly reject them using an allowlist (e.g., `['http:', 'https:']`).
