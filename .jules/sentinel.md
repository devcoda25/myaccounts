## 2024-04-22 - Fix SSRF and XSS in URL Validation
**Vulnerability:** The application was vulnerable to Server-Side Request Forgery (SSRF) and Open Redirects because `isValidUrl` used weak string matching (`.includes('evzone.com')`) which could be bypassed with domains like `evzone.com.attacker.com`. Additionally, `sanitizeUrl` attempted to enforce HTTPS without properly validating the protocol, leading to XSS vulnerabilities when `javascript:` protocols were provided because setting `url.protocol = 'https:'` fails silently on special protocols.
**Learning:** Avoid using string methods like `.includes()` for domain validation; use exact matching (`===`) or `.endsWith()`. Additionally, when using the JavaScript `URL` API, modifying `url.protocol` fails silently for special schemas like `javascript:`.
**Prevention:** Always use explicit allowlists and exact/suffix matching for domains. For URLs, explicitly validate against an allowlist (e.g., `http:`, `https:`) and reject unapproved protocols instead of attempting mutation.
