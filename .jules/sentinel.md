## 2024-05-20 - Enforce Cryptographically Secure Random Number Generation
**Vulnerability:** Weak random number generation using `Math.random()` as a fallback when `window.crypto` is unavailable in `safeRandomBytes`. This exposes sensitive operations like generating WebAuthn challenges, passkeys, internal IDs, and 2FA secrets to predictability, rendering them insecure.
**Learning:** The application mistakenly assumes that failing over to a non-cryptographically secure pseudo-random number generator (PRNG) like `Math.random()` is acceptable for robustness. In security contexts, it's better to fail securely (throw an error) than to proceed with weak randomness, which could compromise the entire system.
**Prevention:** Remove the `Math.random()` fallback in `safeRandomBytes` and any other similar security-sensitive functions. If `window.crypto.getRandomValues` fails or is unavailable, the application MUST throw an error and halt the operation securely, adhering to a "fail closed" security posture.
