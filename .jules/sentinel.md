## 2025-03-05 - URL Validation Bypass via Substring Matching
**Vulnerability:** The application used `String.prototype.includes()` to validate domains in `isValidUrl` and `sanitizeUrl` (e.g., `parsed.hostname.includes('evzone.com')`). This is a critical security flaw because an attacker could bypass validation by hosting their exploit on `evzone.com.evil.com` or passing a dangerous protocol like `javascript:` which bypassed the protocol check if not strictly enforced.
**Learning:** Substring matching (`includes`, `indexOf`) is insufficient for domain validation as it doesn't enforce domain boundaries. Furthermore, URLs must be explicitly restricted to safe protocols (`http:`, `https:`) to prevent execution of malicious code via schemes like `javascript:` and `data:`.
**Prevention:** Always use strict equality (`===`) or strict suffix matching (`endsWith`) for domains (e.g., `parsed.hostname === 'evzone.com' || parsed.hostname.endsWith('.evzone.com')`). Additionally, always explicitly whitelist allowed protocols (`http:`, `https:`) and reject unknown or dangerous ones entirely.
