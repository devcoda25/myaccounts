## 2024-11-20 - URL Domain and Protocol Bypass Vulnerability
**Vulnerability:** `url.ts` in `isValidUrl` used `parsed.hostname.includes('evzone.com')` allowing bypass via domains like `evzone.com.attacker.com`. Additionally, `sanitizeUrl` attempted to enforce HTTPS by modifying `parsed.protocol` without verifying the starting protocol. The JavaScript `URL` API silently ignores protocol modifications for some special schemas like `javascript:`, resulting in a failure to sanitize `javascript:` URLs, leading to potential XSS vulnerabilities.
**Learning:** `.includes()` should never be used to validate domains due to domain spoofing. Modifying properties on the `URL` object fails silently for special schemas (like `javascript:`).
**Prevention:** Always use strict exact matching (`===`) or `.endsWith()` for domain validation. When sanitizing URLs, validate against an explicit allowlist (e.g., `http:`, `https:`) and reject unapproved protocols before attempting any modifications.
