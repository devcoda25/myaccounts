
## 2024-05-29 - Prevent URL Validation Bypasses and XSS in URL Sanitization
**Vulnerability:** The application was vulnerable to SSRF/Open Redirects because `isValidUrl` used `.includes('evzone.com')`, allowing bypasses like `evzone.com.attacker.com`. Additionally, `sanitizeUrl` attempted to mutate the `protocol` property to `https:` for insecure URLs without an allowlist. The `URL` API silently fails to update the protocol for special schemas like `javascript:`, meaning the function returned the original XSS payload unaltered. It also incorrectly threw and returned an empty string for valid relative paths.
**Learning:** String inclusion `.includes()` is dangerous for domain validation. The javascript `URL` API does not mutate special schemas when modifying `url.protocol`, making un-allowlisted protocol mutation an ineffective sanitization strategy.
**Prevention:** Always use strict domain matching (`===` or `.endsWith()`) against an allowed list. Sanitize URLs by explicitly validating against a protocol allowlist (`['http:', 'https:']`) and rejecting unapproved schemas before attempting mutation. Provide a safe fallback mechanism for relative paths by checking `startsWith('/')` and explicitly blocking path traversals or protocol-relative bypasses.
