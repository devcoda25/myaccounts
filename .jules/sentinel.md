## 2024-05-15 - [SSRF & Open Redirect via .includes() Domain Validation]
**Vulnerability:** The application used `url.hostname.includes('evzone.com')` to validate if a URL belonged to a trusted domain. This could be bypassed by an attacker registering a domain like `evzone.com.attacker.com` or `fake-evzone.com`, leading to Server-Side Request Forgery (SSRF) and Open Redirects. Additionally, `sanitizeUrl()` failed to reject XSS schemas like `javascript:` as mutating `protocol` silently fails on certain schemas.
**Learning:** Checking sub-strings for domain validation is dangerous and can lead to complete bypasses. Validating the URL protocol via mutation is inadequate since the `URL` API silently ignores changes for unsupported schemas.
**Prevention:** Always use strict exact matching (`===`) or end matching (`.endsWith('.evzone.com')`) for domains. Reject unauthorized URL schemas (like `javascript:`) before attempting to process the URL, rather than attempting to mutate the protocol property.
