## 2024-03-22 - [Fix SSRF/Open Redirect via `.includes()` URL validation]
**Vulnerability:** The `isValidUrl` and `sanitizeUrl` functions in `src/sanitizers/url.ts` use `.includes('evzone.com')` to validate the hostname. This is vulnerable to Server-Side Request Forgery (SSRF) and Open Redirect because it allows subdomains or domains containing the string, like `evzone.com.attacker.com` or `not-evzone.com`.
**Learning:** Using `.includes()` on domain names is inherently unsafe as it matches any substring. Domain validation must use exact matching (`===`) or suffix matching (`.endsWith()`) ensuring the suffix is a full domain or subdomain (e.g., `endsWith('.evzone.com')` and `=== 'evzone.com'`).
**Prevention:** Always validate domains using strict string equality or secure suffix checking. When using `.endsWith()`, ensure you prepend a dot (`.`) to the domain or explicitly check for exact match to prevent bypassing (e.g., `test-evzone.com` bypassing `endsWith('evzone.com')`).
