## 2024-05-24 - [HIGH] Fix URL validation and sanitization bypasses
**Vulnerability:** The `isValidUrl` and `sanitizeUrl` functions were vulnerable to Open Redirect, SSRF, and XSS. They used a loose `.includes('evzone')` check for validation (allowing `https://evzone.com.attacker.com`) and did not restrict protocols strictly enough in `sanitizeUrl` (allowing `javascript:` protocols). Furthermore, relative URLs were completely stripped in `sanitizeUrl`, potentially breaking application navigation.
**Learning:** Checking for substrings in domain names (via `.includes()`) is insufficient for URL security due to subdomain takeovers and homograph attacks. Also, sanitizing URLs must explicitly block unapproved schemas like `javascript:` as standard URL APIs (`new URL()`) may not reliably alter or strip special schemas on protocol modification.
**Prevention:** Use strictly-defined trusted domains, leveraging array exact matches or strict suffix matches (`.endsWith('.evzone.com')`). Enforce a strict protocol allowlist (e.g. `http:`, `https:`) during sanitization and use safe path filtering when falling back to relative URLs.
